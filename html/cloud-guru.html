Identity Access Management - IAM

    Definition
  • Allow to manage users and their level of access to the AWS Console
    Benefits
  • Centralised control of your AWS account
  • Shared Access to your AWS account
  • Granular Permission
  • Identity Federation (Active Directory, Facebook, Linkein, ...)
  • Multifactor Authentication
  • Provide temporary access for users/devices and services where necessary
  • Allows you to set up your own password rotation policy
  • Integrates with many different AWS services
  • Supports PCI DSS COmpliance
    Critical terms
  • Users: end users
  • Groups: a collection of users under one set of permissions
  • Roles: you create roles and can the assign them to AWS resources
  • Policies: a document that defines one or more permissions, on top of above terms

    Security Token Service (STS)
  • Grant users limited and temporary access to AWS resources
  • There are 3 user sources
    • Federation (Active Directory)
      • Uses Security Assertion Markup Language (SAML)
      • Grants temporary access based off the users Active Directory credentials - not need to be a user in IAM
      • Single sign on (SSO) allows users to log in to AWS console without assigning IAM credentials
    • Federation with Mobile Apps
      • Use Facebook/Amazon/Google or other OpenID provider to log in
    • Cross Account Access
      • Let's users from one AWS account access resources in another
  • Key terms
    • Federation: combining or joining a list of users in one domain (such as IAM) with a list of users in another domain (such as AD, Facebook)
    • Identity Broker: a service that allows you to take an identity from point A and join it to point B
    • Identity Store: Services like AD, Facebook, ...
    • Identities: a user of service like Facebook, ...
  • Senario
    1. Employee enters their username and password
    2. The application calls an Identity Broker, which capture username and password
    3. The Identity Broker uses the orginization's LDAP directory to validate the employee's identity
    4. The Identity Broker calls the new GetFederationToken function using IAM credentials, that includes an IAM policy and a duration
    5. The STS confirms the policy and return 4 values of the app: access key, secret access key, token and duration
    6. The Identity Broker returns the temporary security credentials to the reporting app
    7. The targeted service (such as data storage app) uses the temporary security credentials to make requests to the service
    8. The service uses IAM to verify that the cresentials allow the requested operation on the given instance
    9. IAM provides service with the go-ahead to perform the requested operation

Amazon Elastic Compute Cloud - EC2

    Definition
  • Web service that provides resizable compute capacity in the cloud
  • Provide the tools to build failure resiliet applications and isolate themselves from common failure scenarios
    Options
  • On Demand: pay fixed rate by the hour without commitment
  • Reserved: provides with a capacity reservation and offer significant discount
  • Spot: enable to bid price for instance capacity
  • Dedicated Hosts: physical EC2 server dedicated for your use, not support multi-tenancy or cloud deployments
    Instance Types
  • FIGHT DR MC PX
  • F: FPGA
  • I: IOPS
  • G: Graphics
  • H: High disk throughput
  • T: T2 Micro
  • D: Density
  • R: RAM
  • M: main choice for general purpose apps
  • C: Compute
  • P: Graphics (Pics)
  • X: Extreme Memory
    EBS
  • Allow to create storage volumes and attach them to EC2 instances
  • EBS volumes are placed in a specific AZ => automatically replicated to protect from the failure of a single component
  • Types
      General Purpose SSD (GP2)
    • Ratio of 3 IOPS per GB to 10,000 IOPS
      Provisioned IOPS SSD (IO1)
    • Designed for I/O intensive application such as large relational or NoSQL databases
    • > 10,000 IOPS
      Throughput Optimized HDD (ST1)
    • Big data
    • Data warehouse
    • Log processing
    • Cannot be a boot volume
      Cold HDD (SC1)
    • Lowest cost storage for infrequently accessed workloads
    • File Server
    • Cannot be a boot volume
      Magnetic
    • Lowest cost per GB
    • Bootable
    • Infrequently accessed
  • Upgrade
  • Cannot upgrade standard type
  • Ability to create snapshot that can be used to create volume, image, and copy

  • Security group
    • Is a Virtual Firewall
    • One instance can have multiple security groups
    • All inbound Traffic is blocked by default
    • All outbound Traffic is allowed by default
    • Changes take effect immediately
    • The relationship between EC2s and Security Groups is M:N
    • Security Groups are stateful
    • Cannot block specific IP address using Security Groups, instead use Network Access Control Lists
    • Ability to specify allow rules, but not deny rules
